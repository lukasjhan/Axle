# Ports

The host implements these and injects them via `WalletPorts`. The SDK provides defaults for `Rng`,
`WalletClock`, and `TransactionLogStore` — you must supply at least `SecureArea`, `StorageDriver`,
and `HttpTransport`.

## SecureArea

Creates and uses key material that never leaves the secure boundary. Back it with Android Keystore or
iOS Secure Enclave; a software implementation exists for tests.

<Tabs groupId="lang">
<TabItem value="kotlin" label="Kotlin">

```kotlin
interface SecureArea {
    val id: SecureAreaId
    suspend fun createKey(spec: KeySpec): KeyInfo
    suspend fun publicKey(handle: KeyHandle): EcPublicKey
    // signing is exposed via SecureAreaJwsSigner / SecureAreaCoseSigner
}
```

</TabItem>
<TabItem value="swift" label="Swift">

```swift
public protocol SecureArea {
    var id: SecureAreaId { get }
    func createKey(spec: KeySpec) async throws -> KeyInfo
    func publicKey(key: KeyHandle) async throws -> EcPublicKey
    // signing is exposed via SecureAreaJwsSigner / SecureAreaCoseSigner
}
```

</TabItem>
</Tabs>

## StorageDriver

Byte persistence keyed by collection + key, plus a transaction scope. Wrap it with encryption at rest
for production.

<Tabs groupId="lang">
<TabItem value="kotlin" label="Kotlin">

```kotlin
interface StorageDriver {
    suspend fun put(collection: String, key: String, value: ByteArray)
    suspend fun get(collection: String, key: String): ByteArray?
    suspend fun delete(collection: String, key: String)
    suspend fun keys(collection: String): List<String>
    suspend fun transaction(block: suspend StorageTx.() -> Unit)
}
```

</TabItem>
<TabItem value="swift" label="Swift">

```swift
public protocol StorageDriver: Sendable {
    func put(collection: String, key: String, value: [UInt8]) async throws
    func get(collection: String, key: String) async throws -> [UInt8]?
    func delete(collection: String, key: String) async throws
    func keys(collection: String) async throws -> [String]
    func transaction(_ block: (any StorageTx) async throws -> Void) async throws
}
```

</TabItem>
</Tabs>

## ProximityTransport

A duplex framed-message channel for ISO 18013-5 (see [Proximity](../guides/proximity)). The SDK
drives the message exchange; you supply the BLE/NFC radio. `retrievalMethods()` / `nfcCarrier()` let
the transport tell the SDK its BLE carrier (UUID + mode) for the QR / NFC engagement — both default to
none, so a minimal transport implements only the three duplex methods.

<Tabs groupId="lang">
<TabItem value="kotlin" label="Kotlin">

```kotlin
interface ProximityTransport {
    suspend fun send(message: ByteArray)
    suspend fun receive(): ByteArray
    suspend fun close()
    fun retrievalMethods(): List<ByteArray> = emptyList()  // BLE DeviceRetrievalMethod(s) for QR engagement
    fun nfcCarrier(): NfcCarrier? = null                   // BLE carrier for NFC static handover
}
```

</TabItem>
<TabItem value="swift" label="Swift">

```swift
public protocol ProximityTransport: Sendable {
    func send(_ message: [UInt8]) async throws
    func receive() async throws -> [UInt8]
    func close() async
    func retrievalMethods() -> [[UInt8]]   // BLE DeviceRetrievalMethod(s) for QR engagement
    func nfcCarrier() -> NfcCarrier?        // BLE carrier for NFC static handover
}
```

</TabItem>
</Tabs>
